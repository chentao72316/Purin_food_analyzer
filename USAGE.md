# 使用说明

## 快速开始

### 1. 安装依赖

```bash
npm install
```

### 2. 配置环境变量

创建 `.env.local` 文件：

```env
ARK_API_KEY=d3c412ec-e817-415d-b896-6803f29a639a
ARK_ENDPOINT_ID=ep-20251214002916-lthkj
ARK_API_URL=https://ark.cn-beijing.volces.com/api/v3/responses
```

### 3. 启动开发服务器

```bash
npm run dev
```

访问 http://localhost:3000

## 使用流程

1. **上传图片**
   - 点击上传区域选择文件
   - 或拖拽图片到上传区域
   - 或点击"拍照"按钮（移动端）

2. **开始识别**
   - 上传图片后，点击"开始识别"按钮
   - 等待AI模型分析（通常需要几秒到十几秒）

3. **查看结果**
   - 图片上会显示标注框：
     - 深红色框：高嘌呤食物（>150mg/100g）
     - 红色框：中嘌呤食物（50-150mg/100g）
   - 下方会显示详细的食物信息

4. **重新识别**
   - 点击"重新选择"按钮可以上传新的图片

## 功能说明

### 图片上传
- 支持格式：JPG、PNG、WEBP
- 最大大小：10MB
- 支持拖拽上传
- 支持移动端拍照

### 识别结果
- **高嘌呤食物**：用深红色框标注，显示详细信息
- **中嘌呤食物**：用红色框标注，显示详细信息
- **低嘌呤食物**：在底部简要列出

### 食物信息
每个食物卡片显示：
- 食物名称
- 嘌呤含量（mg/100g）
- 食物详细介绍

## 注意事项

1. **网络连接**：识别需要调用豆包API，确保网络连接正常
2. **图片质量**：清晰的图片能获得更好的识别效果
3. **识别时间**：根据网络和模型负载，识别可能需要几秒到十几秒
4. **API限制**：注意API的调用频率限制

## 故障排除

### 识别失败
- 检查网络连接
- 确认图片格式和大小符合要求
- 查看浏览器控制台的错误信息

### 图片无法上传
- 检查图片格式是否为JPG、PNG或WEBP
- 确认图片大小不超过10MB
- 尝试刷新页面

### API调用失败
- 检查 `.env.local` 文件中的API配置
- 确认API密钥有效
- 检查API服务是否正常

## 技术细节

### 坐标系统
- 使用像素坐标系统
- 坐标格式：{x1, y1, x2, y2}
- 坐标原点(0,0)位于图片左上角

### 嘌呤分类
- 高嘌呤：>150mg/100g
- 中嘌呤：50-150mg/100g
- 低嘌呤：<50mg/100g
